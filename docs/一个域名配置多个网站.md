# 一个域名配置多个网站（单台 VPS 跑多个项目）指南

当你在同一台 VPS 上部署了多个不同的项目（比如：一个 Grok API 后端、一个 Web 管理前端、一个个人博客），你不需要为每个项目购买单独的服务器，也不需要让用户在访问时带上丑陋的端口号（如 `:8000` 或 `:3000`）。

最标准的做法是**使用子域名（Subdomains）搭配反向代理（Reverse Proxy）**。

---

## 核心原理：反向代理

一台 VPS 只有一个公网 IP 地址。当浏览器访问服务器时，HTTP/HTTPS 的默认请求端口是 `80` 和 `443`。

**反向代理软件**（如 Nginx、Caddy）会监听服务器的 `80/443` 端口。当外部请求到达时，代理软件会**读取请求头中的域名信息**，然后根据配置，将流量内部转发（打入）到对应项目绑定的内部端口。

**架构图解：**
```text
用户请求 ---> [ api.yourdomain.com ] ---\
用户请求 ---> [ web.yourdomain.com ] -----> VPS (IP: 172.245.xx.xx)
用户请求 ---> [ blog.yourdomain.com ] ---/       |
                                                 v
                                    [ 反向代理软件 (Nginx/Caddy) 监听 80/443 ]
                                    /            |             \
                (如果域名是 api...) /      (如果是 web...)      \ (如果是 blog...)
                                 /               |               \
                        [内部端口 8000]    [内部端口 3000]    [内部端口 8080]
                        (Grok2API项目)     (Web前端项目)      (WordPress博客)
```

---

## 步骤一：DNS 解析（配置子域名）

首先去你的域名注册商（如 Cloudflare, 阿里云, GoDaddy）的 DNS 设置页面，添加多个 `A` 记录，让它们都指向同一个 VPS 的 IP。

| 类型 (Type) | 主机记录/名称 (Name) | 值/指向 (Target) | 作用 |
|------------|---------------------|------------------|------|
| A | `api` | `172.245.16.80` | API 后端项目（api.yourdomain.com） |
| A | `web` | `172.245.16.80` | Web 前端项目（web.yourdomain.com） |
| A | `blog` | `172.245.16.80` | 博客项目（blog.yourdomain.com） |
| A | `@` | `172.245.16.80` | 根域名（yourdomain.com） |

---

## 步骤二：项目端口规划

确保 VPS 上的每个项目都在**不同的内部端口**上运行。例如：
1. **项目 A (Grok2API)** 运行在 `127.0.0.1:8000`
2. **项目 B (Next.js 前端)** 运行在 `127.0.0.1:3000`
3. **项目 C (其他后端)** 运行在 `127.0.0.1:8080`

> **安全提示**：在 Docker 或本地启动配置中，建议将端口绑定到 `127.0.0.1`（如 `127.0.0.1:8000:8000`），这样可以防止外部通过 `IP:端口` 直接绕过域名和 HTTPS 访问你的项目。

---

## 步骤三：安装并配置反向代理（推荐 Caddy）

对于新手和希望省事的人，强烈推荐使用 **Caddy** 而不是 Nginx。因为 Caddy 会**自动**为你所有的子域名申请并免费续期 SSL（HTTPS）证书，而 Nginx 需要手动配置 Certbot。

### 1. 在 VPS 安装 Caddy (Ubuntu/Debian)
```bash
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy
```

### 2. 编辑 Caddy 配置文件
```bash
nano /etc/caddy/Caddyfile
```

### 3. 配置多个子域名转发
清空默认内容，按照你的项目端口写入以下配置（替换为你真实的域名）：

```text
# 1. 代理 Grok API 项目
api.yourdomain.com {
    reverse_proxy localhost:8000
}

# 2. 代理 Web 前端项目
web.yourdomain.com {
    reverse_proxy localhost:3000
}

# 3. 代理博客项目
blog.yourdomain.com {
    reverse_proxy localhost:8080
}

# 4. (可选) 代理根域名
yourdomain.com {
    reverse_proxy localhost:3000
}
```

保存文件后，重启 Caddy 应用最新配置：
```bash
systemctl reload caddy
```

**就这么简单！** Caddy 会在后台自动为这三个子域名去 Let's Encrypt 申请 HTTPS 证书。等几秒钟后，你就可以直接在浏览器通过 `https://api.yourdomain.com`、`https://web.yourdomain.com` 访问不同的项目了。

---

## 步骤四：修改各项目的环境变量 / 配置文件

当你的项目被挂在子域名后面时，部分项目需要知道自己的“公网真实地址”以便正确拼接返回给用户的链接、处理跨域 (CORS) 或 OAuth 登录。

以 Grok2API 为例，打开它的 `setting.toml`：
```toml
[app]
app_url = "https://api.yourdomain.com"
```
重启 Docker 容器。现在 API 内部生成的视频或图片链接，就会自动带上漂亮的 `https://api.yourdomain.com` 前缀。

对于前端项目（比如 Next.js/Vue），你也可能需要更新类似 `NEXT_PUBLIC_API_URL=https://api.yourdomain.com` 的环境变量。

## 附录：如果你坚持使用 Nginx

如果你更喜欢传统的 Nginx，你需要手动做更多配置：

1. 安装 Nginx 和 Certbot:
   ```bash
   apt install nginx certbot python3-certbot-nginx
   ```
2. 在 `/etc/nginx/sites-available/` 下为每个子域名创建一个配置文件：
   ```nginx
   # /etc/nginx/sites-available/api.yourdomain.com
   server {
       listen 80;
       server_name api.yourdomain.com;
       
       location / {
           proxy_pass http://127.0.0.1:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```
3. 启用配置并重启 Nginx:
   ```bash
   ln -s /etc/nginx/sites-available/api.yourdomain.com /etc/nginx/sites-enabled/
   systemctl restart nginx
   ```
4. 手动申请 HTTPS 证书:
   ```bash
   certbot --nginx -d api.yourdomain.com
   ```
*(以上 Nginx 配置需对每个子域名重复一次)*
